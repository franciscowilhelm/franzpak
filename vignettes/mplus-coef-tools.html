Formatting Mplus Coefficients with franzpak

Introduction

This vignette demonstrates the Mplus coefficient tools in franzpak:

  • coef_wrapper() – pulls coefficients from an MplusAutomation model and splits the Mplus parameter labels into dependent (DV) and independent (IV) variable columns.
  • coef_table_mplus() – arranges those coefficients into a publication-ready, predictor x outcome table (rendered with gt when available).

Both functions:

  • auto-detect Bayesian vs. frequentist estimation and adapt the display (estimate + SE with p-value stars, or estimate + 95% interval with interval-exclusion stars);
  • understand two-level (multilevel) models, splitting effects into Within and Between sections and reporting the means/intercepts of random slopes as the between-person means of within-person effects. When a model has random slopes, coef_table_mplus() returns a pair of tables – the fixed-effects table plus a dedicated Random effects table.

The examples use the classic Mplus User’s Guide data files that ship with franzpak in inst/extdata/.

library(franzpak)
library(MplusAutomation)

# Locate a bundled .out file whether the package is installed or loaded from
# the source tree (e.g. via devtools::load_all()).
mplus_file <- function(name) {
  p <- system.file("extdata", name, package = "franzpak")
  if (nzchar(p)) return(p)
  cands <- c(file.path("inst", "extdata", name),
             file.path("..", "extdata", name),
             file.path("..", "..", "inst", "extdata", name))
  hit <- cands[file.exists(cands)]
  if (length(hit) == 0) stop("Could not locate ", name)
  hit[[1]]
}

Part 1: A single-level model

ex5.11 is a single-level structural equation model: two exogenous factors (F1, F2) predict a mediator factor (F3), which in turn predicts an outcome factor (F4).

m_sem <- readModels(mplus_file("ex5.11.out"), quiet = TRUE)

coef_wrapper(): tidy coefficients

By default coef_wrapper() returns the unstandardized regression parameters with the Mplus DV<-IV label split into separate columns.

coef_wrapper(m_sem)

Add confidence intervals with addci = TRUE (these come from the Mplus CINTERVAL output):

coef_wrapper(m_sem, addci = TRUE)

Request a different coefficient scale with type – e.g. fully standardized (STDYX) estimates:

coef_wrapper(m_sem, type = "stdyx")

coef_table_mplus(): a publication table

coef_table_mplus() reshapes the coefficients into a predictor x outcome table. For a frequentist model the default display is estimate + standard error with stars for p < 0.05.

Part 2: A two-level model with a random slope

ex9.2c is a two-level regression (Mplus User’s Guide example 9.2c). The model is:

%WITHIN%
  s | y ON x;        ! random slope of y on x (a within-person effect)
%BETWEEN%
  y s ON w x;        ! between-level regressions of y and of the slope s
  y WITH s;

The key feature is the random slope s | y ON x: the effect of x on y within clusters is allowed to vary between clusters. Its intercept is the average within-person slope – a between-person summary of a within-person effect.

m_ml <- readModels(mplus_file("ex9.2c.out"), quiet = TRUE)

coef_wrapper() annotates the multilevel structure

For two-level models coef_wrapper() adds several annotation columns. Here we request regression, expectation and variability parameters so that intercepts/means and the random-slope variance are all included:

coef_wrapper(m_ml, params = c("regression", "expectation", "variability"))
  • level – the Mplus section the parameter belongs to (Within / Between).
  • random_slope – TRUE for any parameter that pertains to the random slope S (its mean, its predictors, or its variance).
  • within_avg_effect – TRUE for the mean/intercept of a random slope, i.e. a between-person mean of a within-person effect.
  • random_effect – TRUE for the slope’s other parameters: its cross-level predictors and its (residual) variance.
  • variance – TRUE for (residual) variance parameters.
  • display_level – the table section the row belongs to: Within, Between, or Random effects.

coef_wrapper() re-expresses each slope parameter in terms of the within-person relationship the slope captures (s | y ON x):

  • the mean of S (1.017) becomes the average within-person effect of x on y, so it is reported with DV = "Y", IV = "X" and display_level = "Within";
  • the cross-level regressions S ON W and S ON X describe how the slope varies across clusters; they move to display_level = "Random effects" and are reported under the slope’s outcome (DV = "Y"), keeping the between predictor;
  • the slope’s (residual) variance also moves to Random effects, labelled IV = "Residual variance".

Variances that do not belong to a random slope (here the residual variances of Y) are kept by coef_wrapper() but omitted from the table below.

coef_table_mplus() returns a fixed-effects table and a Random effects table

A random slope is associated with a specific within-person fixed effect and only secondarily with an outcome variable, so cramming all of its parameters into the predictor x outcome grid (under the Y column) made the table hard to read. Instead, when a model has random slopes coef_table_mplus() returns a named list with two elements:

  • $fixed – the main predictor x outcome table, with Within and Between sections only;
  • $random – a Random effects table listing every random-slope parameter.

The fixed-effects table keeps two sections:

  • Within – within-person effects. The within-person effect of X on Y (the random-slope mean) appears as predictor X under outcome Y, marked with a superscript letter (the APA convention for a specific note; the dagger is avoided because APA reserves it for marginal significance).
  • Between – between-person effects on the real outcome Y. The between-person effect of X on Y (1.024) is a separate, fixed coefficient, kept clearly apart from its within-person counterpart (1.017). Cross-level interactions (the slope S regressed on a between predictor) stay here too, relabelled to name the interaction: S ON W becomes W (between) x X (within) (the between variable W moderating the within X -> Y slope).

The Random effects table collects the slope’s parameters, grouped by slope and ordered predictors -> mean/intercept -> variance. A slope’s mean/intercept and variance are relabelled by slope (S Intercept, S Residual Variance), while its cross-level predictors keep their raw Mplus label (e.g. S<-W) so their source stays unambiguous:

Where the intervals come from

coef_table_mplus() (via an internal helper) first looks for interval columns in the main estimate table – the credibility limits Bayesian runs print inline in MODEL RESULTS. These are reconstructed from the same parameter columns that supply the estimates, so the labels always line up and the values join cleanly even when variable names are long. Only if the estimate table carries no intervals (typical of frequentist runs) does it fall back to MplusAutomation’s confint(), which reads the dedicated CINTERVAL table. If no intervals can be found at all (a frequentist run without CINTERVAL), the table gracefully falls back to the estimate + SE display.

Preferring the estimate table also sidesteps an Mplus quirk: variable names are truncated to 8 characters in the CINTERVAL section, so a long name like M_APREG_S is printed there as M_APREG_. Joining intervals from that section by label would silently drop the affected rows; reading from the (untruncated) estimate table avoids the problem.

Part 3: A two-level model with several random slopes

ex9.5 (Mplus User’s Guide example 9.5, Bayesian variant) is a two-level path analysis with two random slopes, so it exercises the same machinery as Part 2 but with more than one slope in play:

%WITHIN%
  s2 | y2 ON y1;     ! random slope of y2 on y1
  y2 ON x2;          ! fixed within-person effect
  s1 | y1 ON x2;     ! random slope of y1 on x2
  y1 ON x1;          ! fixed within-person effect
%BETWEEN%
  y1 y2 s1 s2 ON w;  ! between-level regressions of the outcomes and both slopes

The two slopes capture different within-person relationships – s2 is the effect of y1 on y2, and s1 is the effect of x2 on y1 – so each is re-expressed under its own outcome (Y2 and Y1 respectively).

m_ml2 <- readModels(mplus_file("ex9.5_BAYES.out"), quiet = TRUE)

coef_wrapper() classifies the parameters of both slopes independently. The latent slope names S1/S2 never reach the output: each slope’s mean, its cross-level predictor W, and its residual variance are remapped to the within-person relationship it represents.

coef_wrapper(m_ml2, params = c("regression", "expectation", "variability"))

In the fixed-effects table, the means of s2 (the within-person effect of y1 on y2) and s1 (the within-person effect of x2 on y1) appear in the Within section marked with a superscript letter. Each slope’s cross-level predictor W becomes a relabelled interaction in Between (W (between) x Y1 (within) under Y2, W (between) x X2 (within) under Y1):

Part 4: Growth models and the | operator

The | operator serves multiple roles in Mplus. It declares random slopes (Parts 2–3), but also growth factors and latent variable interactions (XWITH). The package distinguishes these cases conservatively:

  • growth-factor declarations such as i s | y1-y4 AT ... trigger the growth-specific table layout automatically; use model_type = "growth" to force it, or model_type = "default" for the ordinary predictor x outcome layout;
  • ON-form random slopes are still handled by the two-level fixed/random table machinery in Parts 2–3;
  • XWITH interactions are not random effects – the interaction term is an ordinary predictor – so they are detected and excluded from the random-effect handling (Part 5).

ex6.12 is a single-level growth model with TYPE = RANDOM: two growth factors (i, s) declared with |, a random slope st for a time-varying covariate, and all three regressed on x:

i s | y1-y4 AT a11-a14;   ! intercept & slope growth factors
st | y1 ON a21;           ! random slope of a time-varying covariate
st | y2 ON a22;
st | y3 ON a23;
st | y4 ON a24;
i s st ON x;              ! growth factors / slope regressed on x
m_growth <- readModels(mplus_file("ex6.12.out"), quiet = TRUE)

The slope detector recognises every | declaration, tagging the growth factors kind = "growth" (no outcome/predictor decomposition) and the random slope st kind = "slope":

franzpak:::mplus_random_slopes(m_growth)

Because this model is single-level (no %WITHIN% / %BETWEEN% split), the multilevel re-expression used in Parts 2–3 does not apply. coef_wrapper() returns plain coefficients with each growth factor / slope keeping its own name as the outcome – the regression DVs are no longer corrupted by the random-slope remapping:

coef_wrapper(m_growth)

coef_table_mplus() detects the growth-factor declaration and returns a growth-specific table. The predictors of the factors are shown first, followed by dedicated sections for factor means and variances:

Part 5: Latent variable interactions (XWITH)

ex5.13 is a single-level SEM with a latent interaction. The | here defines an interaction term f1xf2 between two latent variables, which then predicts f3:

f4 ON f3;
f3 ON f1 f2;
f1xf2 | f1 XWITH f2;   ! latent interaction term (NOT a random effect)
f3 ON f1xf2;
m_xwith <- readModels(mplus_file("ex5.13.out"), quiet = TRUE)

XWITH is detected by the | parser and excluded from the random effects, so the helper returns an empty tibble:

franzpak:::mplus_random_slopes(m_xwith)

The interaction term is kept as an ordinary predictor. F1XF2 appears as a predictor of F3, untouched by any random-effect remapping:

coef_wrapper(m_xwith, params = "regression")
